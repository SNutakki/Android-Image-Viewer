Image Crawler Fork-Join Pool App (Assignment 4a)

The purpose of this assignment is to deepen your experience using Java 8 functional programming features and their integration with the Java fork-join pool. In particular, in this assignment you'll develop an app that uses the Java 8 fork-join framework to obtain, store, and transform images.

As before, the code you'll write for this assignment will have GUI and will run as an app. Please use this GUI to help debug your solution.

Getting Started

Overview of the Assignment 4a

In this part of the assignment you'll complete the implementation of an image crawler class in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/ForkJoinCrawler.java file that obtains, stores, and transforms images. This class uses Java fork-join pool framework to perform an "image crawl" starting from a root Uri. Depending on the parameters used to run the tests, images from HTML page reachable from the root Uri are either downloaded from a remote web server or read from the local file system and the results are stored in files.

The bulk of the code for the app is provided in the skeletons. In fact, a complete solution using Java 7 features provided in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/crawlers/SequentialLoopsCrawler.java file. Your will therefore focus on converting this Java 7-based class to the ForkJoinCrawler class that uses the Java fork-join pool.

Everyone taking the course (i.e., both ugrads and grads) must also include your completed implementations of all the methods in the image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/ArrayCollector.java and image-crawler/src/main/java/edu/vanderbilt/imagecrawler/utils/Array.java files. Make sure that these classes work properly or you'll lose points.

Your solution can either use conventional for Java 7 for-loop loops or Java 8 lambda expressions and method references, as you see fit.

Unit Tests

JUnit unit tests in the image-crawler/src/test/java/edu/vanderbilt/imagecrawler folder are provided to increase our confidence that your implementation is working as expected. You can run all the unit tests by right-clicking on 'AllTests' in this folder.

If you need help learning how to use the Android Studio GUI to run the JUnit test locally on your computer please see here and/or ask questions. As usual, testing only demonstrates the presence of bugs, not their absence, so don't rely solely on the JUnit tests to detect problems in your code.
